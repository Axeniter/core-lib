# MVVM
MVC - архитектурный паттерн, разделяющий структуру программы на 3 слабосвязанных слоях:
1. Model - бизнес логика приложения
2. View - обрабатывает пользовательский ввод и отображает данные. Привязан к ViewModel, тоесть подписан на событие об изменении ViewModel.
3. ViewModel - обеспечивает связь между Model и View. Управляет состоянием View, обрабатывает пользовательские действия, подготавливает Model для отображения во View, содержит комманды, которыми может пользоваться View для влияния на Model.

Слои связаны между собой с помощью data-binding - привязки, которая автоматически уведомляет связанные слои о своем изменении. Для этого используются реактивные свойства (reactive property).

Паттерн часто используется в десктопных и мобильных приложениях.
