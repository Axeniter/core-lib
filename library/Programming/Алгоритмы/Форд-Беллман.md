# Алгоритм Форда-Беллмана
## Определение
Алгоритм поиска кратчайшего пути из одной вершины в ориентированном графе с произвольными весами (включая отрицательные). Преимущество алгоритма в способности обнаруживать отрицательные циклы.
## Идея
Проводится $|V|-1$ релаксаций рёбер графа, где $|V|$ - количество вершин. Если после этого можно сделать ещё одну релаксацию, то существует отрицательный цикл, и минимальное расстояние до точек, достижимых из отрицательного цикла неопределено (так как при отрицательном цикле можно расстояние уменьшать бесконечно).
## Формально:
$V$ - множество вершин, $E$ - множество рёбер.
$n=|V|$, $m=|E|$.
$s \in V$ - стартовая вершина.
$d[v]$ - текущая длина кратчайшего пути из стартовой вершины $s$ в $v$.
$w(u,v)$ - вес ребра из $u$ в $v$.
Шаги:
1. Инициализация:
    - $d[s]=0$ - расстояние до стартовой вершины
    - $d[v]=+\infty$ для всех остальных вершин
2. Релаксация рёбер (повторяется $n-1$ раз):
	Для каждого ребра $(u,v)\in E$ обновляем расстояние до $v$:	$d[v]=\min(d[v],d[u]+w(u,v))$
3. Проверка на отрицательный цикл:
	Если после $n-1$ релаксаций сделать ещё одну и найдётся хотя-бы одно ребро, для которого $d[u]+w(u,v)<d[v]$, то в графе есть отрицательный цикл, достижимый из $s$.
Псевдокод:
```
for i = 1 to |V| - 1:  
    for each edge (u, v) in E:  
        if d[v] > d[u] + w(u, v):  
            d[v] = d[u] + w(u, v)  

for each edge (u, v) in E:  
    if d[v] > d[u] + w(u, v):  
        return "отрицательный цикл"  
```
