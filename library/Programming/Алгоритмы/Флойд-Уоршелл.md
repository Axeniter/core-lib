# Алгоритм Флойда-Уоршелла
## Определение
Алгоритм динамического программирования для нахождения кратчайших путей между всеми парами вершин во взвешенном графе, где веса рёбер графа могут быть как положительными так и отрицательными, но не должно быть отрицательных циклов. В результате работы алгоритма получаем матрицу расстояний между всеми вершинами.

Временная сложность - $O(V^3)$
Пространственная сложность - $O(V^2)$
где $V$ - количество вершин
## Алгоритм
1. Инициализация:
	Дан граф с $n$ вершинами, который задаётся матрицей весов $w_{ij}$.
	Создаётся матрица $D_{0}$ размера $n \times n$, где:
	- $D_{0}=0$, если $i=j$
	- $D_{0}=w[i][j]$, если ребро $(i,j)$ существует
	- $D_{0}=+\infty$, иначе
2. Динамическое обновление:
	Для каждой промежуточной вершины $k=1,\dots,n$ Обновляется матрица $D_{k}$ (индекс $k$ у матрицы просто показывает текущую промежуточную вершнину, все обновления происходят в одной и той же матрице):
	сравниваем текущий путь из $i$ в $j$, с путём проходящим через вершину $k$, тоесть с путём из $i$ в $k$ + $k$ в $j$: $D_{k}[i][j]=\min(D_{k-1}[i][j],D_{k-1}[i][k]+D_{k-1}[k][j])$
3. Проверка на отрицательные циклы:
	Если после всех итераций, найдётся вершина $i$ такая, что $D[i][j]<0$, то в графе есть отрицательный цикл.

Псевдокод:
```
for k = 1 to n:  
    for i = 1 to n:  
        for j = 1 to n:  
            D[i][j] = min(D[i][j], D[i][k] + D[k][j])  
 
for i = 1 to n:  
    if D[i][i] < 0:  
        return "есть отрицательный цикл"
```
