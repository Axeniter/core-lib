# Бинарный поиск
## Определение
Бинарный поиск - алгоритм поиска элемента в отсортированном массиве. Ключевое условие для использования алгоритма - отсортированный массив.

Временная сложность: $O(\log n)$ - на каждом шаге размер области поиска уменьшается вдвое.
Пространственная сложность: $O(1)$ - не требует дополнительной памяти.
## Принцип
1. Инициализация:
	- имеется отсортированный (по возрастанию) массив `array` с $n$ элементами.
	- Определяем начальные границы поиска: `left = 0`, `right = n-1`.
	- `target` - искомый элемент
2. Цикл:
	Пока `left <= right`:
		Находим `mid = (left + right) / 2`, где `/` - целочисленное деление.
		Сравниваем `target`:
		Если `target == array[mid]` - элемент найден на индексе `mid`
		Если `target < array[mid]` - искомый элемент находится в левой половине, сдвигаем правую границу: `right = mid - 1`
		Если `target > array[mid]` - искомый элемент находится в правой половине, сдвигаем левую границу: `left = mid + 1`
3. Завершение:
	Если границы пересеклись `left > right`, то элемент в массиве отсутствует.
