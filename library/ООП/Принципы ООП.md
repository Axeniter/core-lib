# ООП
## Парадигма
Объектно ориентированное программирование - парадигма программирования, основанная на представлении программы как набор взаимодействующих объектов, каждый из которых является экземпляром определённого класса. Класс - это чертёж, описывающий структуру объекта. Каждый объект имеет данные (поля) и поведение (методы).
## Принципы
### Инкапсуляция
Инкапсуляция - принцип разделения взаимодействия с объектом (public) от его описательной части (private).

Суть принципа в объединении данных и методов, которые работают с этими данными, в одной сущности (в одном классе), и сокрытии внутренней реализации от внешнего использования. Взаимодействие с объектом происходит через публичные методы, представленные самим классом. Внутреннее состояние объекта нельзя изменить напрямую, только через публичные методы.

Ключевые слова:
- public - доступ к члену класса имеет кто угодно извне.
- private - доступ к члену класса имеет только сам класс.
- protected - доступ к члену класса имеет только сам класс и все его производные классы.

Плохой пример:
```cs
public class Wallet
{
	public int Money;
	public string Owner;
	
	public Wallet(int money, string owner)
	{
		Money = money;
		Owner = owner;
	}
}
```
Хороший пример:
```cs
public class Wallet
{
	public int Money { get; private set; }
	public string Owner { get; }
	
	public Wallet(int money, string owner)
	{
		Money = money;
		Owner = owner;
	}
	
	public void Pay(int amount)
	{
		if (Money < amount || amount < 0) return;
		Money -= amount;
	}
	
	public void Take(int amount)
	{
		if (amount < 0) return;
		Money += amount;
	}
}
```
### Наследование
Наследование - принцип создания производных классов на основе существующих.

Суть принципа в создании нового класса, полностью наследующего функционал существующего и при этом добавляющего дополнительный функционал, тем самым расширяя базовый класс.

Пример:
```cs
public class Employer
{
	public string Name { get; }
	public int Salary { get; }
}

public class Manager : Employer
{
	public void Manage()
	{
		...
	}
}
```
### Полиморфизм
Полиморфизм - возможность одного интерфейса иметь различные формы реализации.

Суть принципа в том, что язык позволяет писать логику для некоторых объектов x, где тип x не определён.

Разновидности:
1. Перегрузка
```cs
public int Sum(int x, int y)
{
	return x + y;
}

public float Sum(float x, float y)
{
	return x + y;
}
```
2. Переопределение
```cs
public class Animal
{
	public virtual void Say()
	{
		Console.WriteLine("Im animal");
	}
}

public class Donkey : Person
{
	public override void Say()
	{
		Console.WriteLine("Ia ia");
	}
}
```
3. Шаблон
```cs
class Vector<T> // T - неопределенный тип
{
	public T first;
	public T second;
}
```
