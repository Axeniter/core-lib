# Красно-чёрное дерево
## Определение
Красно-чёрное дерево - это самобалансирующееся бинарное дерево поиска (BST).
## Операции
- Поиск - $O(\log n)$
- Вставка - $O(\log n)$
- Удаление - $O(\log n)$
## Принцип работы
Дерево должно удовлетворять условиям:
1. Каждый узел может быть либо красным, либо чёрным.
2. Корень - чёрный
3. Конечные NULL узлы (листья) - чёрные
4. Каждый красный узел должен иметь только черные дочерние узлы. При этом черные узлы могут иметь дочерние узлы любого цвета
5. Все пути от узла к его листьям должны содержать одинаковое число чёрных узлов. Число черных вершин на произвольном пути от корня до листа называется черной высотой.

Все листья являются по сути фиктивными, поэтому можно создать один NULL узел и ссылаться на него в конце каждого пути.

Высота крачно-черного дерева может быть $\leq 2\log_{2}(n+1)$

Обозначения:
- Отец (родитель) элемента - узел, который ссылается на данный элемент
- Дочерний элемент (потомок, сын) - один из двух элементов, на который ссылается узел
- Дед элемента - отец отца элемента
- Дядя элемента - второй дочерний элемент деда данного элемента (брат отца элемента)
```
      Дед
     /   \
   Отец   Дядя
  /
Сын
```
## Вставка элемента
Каждый элемент вставляется вместо листа, поэтому для выбора места вставки спускаемся от корня пока не встретим NULL-узел. Вставляем вместо него новый элемент окрашенный в красный и с листами (NULL-узлами) в качестве потомков.

Если вставленный элемент является корнем - перекрашиваем его в чёрный.

Проверка балансировки:
1. Если отец элемента окрашен в черный, то свойства дерева не нарушены, балансировка не требуется.
2. Если отец окрашен в красный, то есть два случая:
Дядя красный:
```
    B
   / \
  R   R
 /
R
```
Перекрашиваем отца и дядю в черный цвет, а деда - в красный.

Далее:
- Если дед стал корнем - красим в черный
- Если родитель деда красный, повторяем балансировку.

Дядя черный:
```
    B
   / \
  R   B
 /
R
```
В этом случае нужно выполнить поворот
