# Модели разработки
## Каскадная (Waterfall)
Waterfall - классическая модель разработки ПО, представляющая собой последовательный и линейный процесс, в котором каждый этап разработки выполняется после полного завершения предыдущего.

Модель подходит для проектов с четко определенными требованиями, где изменения не ожидаются и для небольших или средних проектов, где изменения в требованиях или риски минимальны.

Плюсы:
- Простота использования
- Четкая документация
- Легкость в управлении
- Легко оценить сроки
Минусы:
- Негибкая
- Риски выявления ошибок
- Неэффективность для сложных проектов
## V-образная
Модель разработки ПО, которая является расширением каскадной модели. Она визуализирует процесс разработки в форме буквы V, где одна сторона представляет этапы разработки, а другая - этапы тестирования. Модель акцентирует внимание на важности тестирования на каждом этапе разработки.

Подходит для проектов с четкими требованиями, где не ожидаются изменения и для маленьких или средних проектов.

Плюсы:
- Четкая структура
- Тестирование на каждом этапе
- Легкость планирования и оценивания
- Четкая документация
Минусы:
- Низкая гибкость
- Риски выявления ошибок
- Неэффективность для сложных проектов
## Инкрементная
Подход при котором проект разрабатывается и выпускается по частям, называемым инкрементами. Каждый инкремент добавляет новую функциональность к существующей системе, и по мере разработки пользователи получают возможность использования рабочей версии.

Этапы:
- Планирование
- Проектирование
- Разработка первого инкремента с базовой функциональностью
- Тестирование
- Разработка последующих инкрементов выполняя шаги 2-4 для каждого

Плюсы:
- Возможность быстро доставить работающий продукт пользователям
- Гибкость и адаптивность
- Повышение качества на каждом этапе
Минусы:
- Сложность интеграции новых инкрементов в существующую систему
- Зависимость от первоначальных требований
## Итеративная
Модель представляет собой подход, при котором проект разбивается на несколько итераций. Каждая итерация включает в себя все основные этапы разработки: планирование, проектирование, разработка, тестирование и оценка. Итеративный подход позволяет командам регулярно получать обратную связь и вносить изменения на основе результатов предыдущих итераций.

Плюсы:
- Гибкость и адаптивность
- Постоянная обратная связь
- Улучшение качества на каждом этапе
Минусы:
- Необходимость постоянной обратной связи
- Сложность управления
## Спиральная
Модель является итеративным и инкрементным подходом, который акцентирует внимание на управлении рисками. Она визуализируется в виде спирали, где каждый виток является итерацией, включающей этапы планирования, анализа рисков, разработки и оценки. Эффективна для крупных проектов, где требования часто меняются.

Характеристики:
- Каждый виток включает в себя повторение всех этапов разработки и тестирования
- Каждый виток начмнаеися с анализа рисков
- Каждый виток приводит к созданию инкремента продукта, который можно оценить и протестировать
