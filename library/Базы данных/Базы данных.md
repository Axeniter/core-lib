# Базы данных
## Определение
База данных (БД) - совокупность структурированных данных, хранящихся по определенным правилам в электронном виде.

Система управления базами данных (СУБД) - програмное обеспечение для создания и взаимодействия с БД
## Типы баз данных
### Реляционные (SQL)
Данные организованы в виде таблиц (отнощений). Они состоят из:
- Столбцы (атрибуты): определяют какая информация хранится, у каждого столбца есть строгий тип данных.
- Строки (записи): конкретные экземпляры данных, каждая строка - это конкретный объект.

Связи между таблицами устанавливаются с помощью ключей:
- Первичный ключ (primary key): уникальный идентификатор каждой строки в таблице.
- Внешний ключ (foreign key): столбец в одной таблице, который ссылается на первичный ключ в друго таблице.

Чтобы работать с данными, используется SQL (Structured Query Language).

Транзакцией в реляционной базе данных называется последовательность SQL-операций, которая выполняется как единое целое.

Принципы ACID:
- A (Atomicity) - Атомарность: транзакция выполняется целиком или не выполняется вовсе. Нет промежуточных состояний. 
- C (Consistency) - Согласованность: данные всегда переводятся из одного корректного состояния в другое. Все правила (ограничения, ключи) соблюдаются.
- I (Isolation) - Изолированность: параллельные транзакции не мешают друг другу. Как будто они выполняются последовательно.
- D (Durability) - Долговечность: Если транзакция завершилась успешно, её результат навсегда записан в систему, даже при сбое питания.
### Нереляционные (NoSQL)
#### Документные
Данные хранятся в виде документов в разных форматах (JSON, XML и т.д.). Каждый документ - это самостоятельная единица со своей структурой.

Структура - коллекции документов, где документы в одной коллекции могут иметь разную структуру, поля для документов можно добавлять динамически.
#### Графовые
Данные хранятся в виде узлов (сущности) и ребер (связей).

Структура - граф, акцент ставится на связь между данными.
#### Ключ-значение
Данные хранятся в виде ассоциативного массива, где каждый уникальный ключ связан со значением любого типа.
